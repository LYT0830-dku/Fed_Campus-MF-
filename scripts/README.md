repository layout and entrypoints. The scripts documented here are the stable
interfaces expected by CI and release checks.

## Maintained Scripts

### `check_local_assets.sh`

Prints the actual model/data asset directories that the repo will use on the current machine.

```bash
export MFT_MODEL_ROOT=/path/to/mft-models
export MFT_DATA_ROOT=/path/to/mft-data
bash scripts/check_local_assets.sh
```

### `check_release_tree.sh`

Runs source-distribution checks for maintained entrypoints:

- no personal absolute paths in release-facing files;
- no large source files outside ignored asset/build areas;
- package config templates and asset docs are present;
- maintained shell scripts pass `bash -n`.

```bash
bash scripts/check_release_tree.sh
```

### `check_lm_alignment_fixture.sh`

Runs the real-weight `AutoModelAlignment` CTest target against one fixture
generated by `generate_lm_alignment_fixture.py`.

```bash
python3 scripts/generate_lm_alignment_fixture.py \
  --model-dir /path/to/hf-model-snapshot \
  --output runs/model_alignment/model_alignment_fixture.json

bash scripts/check_lm_alignment_fixture.sh \
  runs/model_alignment/model_alignment_fixture.json
```

The script configures `operator/build` when needed, builds
`test_auto_model_alignment`, and passes `MFT_LM_ALIGNMENT_FIXTURE` to CTest.
For padded batch coverage, pass multiple `--prompt` values and `--max-length`
to `generate_lm_alignment_fixture.py`; the generator uses the same label policy
as `make_causal_lm_batch`, including the left-padding boundary mask.

### `generate_peft_lora_step_fixture.py`

Generates a PyTorch/Transformers/PEFT one-step LoRA training fixture for
family-level optimizer-step alignment. The fixture records:

- tokenized `input_ids`, `attention_mask`, and shifted labels;
- LoRA hyperparameters and target modules;
- initial LoRA tensors in MF-consumable layout;
- PyTorch/PEFT loss and LoRA tensor checksums after one Adam step.

For deterministic MF parity, the generator disables stochastic base-model
dropout by default before attaching LoRA. Requested LoRA dropout still follows
`--dropout`. Pass `--keep-model-dropout` only when comparing against a backend
that implements the same family-level dropout behavior.

```bash
python3 scripts/generate_peft_lora_step_fixture.py \
  --model-dir /path/to/hf-model-snapshot \
  --prompt "MobileFineTuner PEFT alignment prompt." \
  --max-length 64 \
  --rank 8 \
  --alpha 16 \
  --lr 2e-4 \
  --target-module q_proj \
  --target-module k_proj \
  --target-module v_proj \
  --target-module o_proj \
  --output runs/model_alignment/peft_lora_step_fixture.json
```

This script requires Python `transformers` and `peft`. It writes to ignored
`runs/` paths because real fixtures contain local model paths and can include
large LoRA tensor arrays.

After generating a fixture, run the C++ parity gate:

```bash
bash scripts/check_peft_lora_step_fixture.sh \
  runs/model_alignment/peft_lora_step_fixture.json
```

The checker builds `test_auto_model_peft_step_alignment` and runs the
`AutoModelPEFTStepAlignment` CTest target with `MFT_PEFT_LORA_STEP_FIXTURE`.
Without a fixture, the CTest target skips; with a fixture, it must match the
recorded PyTorch/PEFT loss and post-Adam LoRA tensors.

### `run_training_smoke.sh`

Runs the repo-level five-model synthetic smoke suite.

```bash
bash scripts/run_training_smoke.sh
```

Environment overrides:
- `SMOKE_STEPS` to control the number of synthetic update steps

### `run_training_real_assets.sh`

Runs a one-step real-asset validation for all five training entrypoints using auto-discovered local model/data directories.

```bash
bash scripts/run_training_real_assets.sh
```

Environment overrides:
- `REAL_STEPS` to control the number of optimizer steps
- `SEQ_LEN` to control the short sanity-check sequence length
- `MFT_MODEL_ROOT` and `MFT_DATA_ROOT` to provide external assets

### `generate_tokenizer_hf_golden_fixtures.py`

Generates HuggingFace tokenizer standard-answer JSONL fixtures for strict C++
tokenizer alignment checks. It only needs tokenizer/config assets, not model
weights.

```bash
export MFT_MODEL_ROOT=/path/to/mft-models
python3 scripts/generate_tokenizer_hf_golden_fixtures.py \
  --output runs/tokenizer_golden/hf_tokenizer_golden.jsonl
```

Llama 3.x tokenizer gates are optional because many machines do not have a
local Llama snapshot. Run them explicitly when a local ByteLevel-BPE
`tokenizer.json` snapshot is available:

```bash
export LLAMA_MODEL_DIR=/path/to/Llama-3.2-1B-Instruct
python3 scripts/generate_tokenizer_hf_golden_fixtures.py \
  --models llama_3_2_1b_instruct \
  --output runs/tokenizer_golden/llama_3_2_1b_hf_tokenizer_golden.jsonl
```

Run the matching CTest target with:

```bash
MFT_TOKENIZER_GOLDEN_JSONL=runs/tokenizer_golden/hf_tokenizer_golden.jsonl \
ctest --test-dir operator/build --output-on-failure -R TokenizerHFGolden
```

Without `MFT_TOKENIZER_GOLDEN_JSONL`, the C++ test reports a skip and does not
require local model assets.

### `generate_lm_alignment_fixture.py`

Generates a PyTorch/Transformers standard-answer fixture for real-model
alignment. The matching C++ gate loads the same local model through
`AutoModelForCausalLM` and checks tokenizer IDs, shifted causal-LM loss, and
last-token logits top-k.

```bash
python3 scripts/generate_lm_alignment_fixture.py \
  --model-dir /path/to/hf-model-snapshot \
  --prompt "MobileFineTuner alignment prompt." \
  --output runs/model_alignment/model_alignment_fixture.json

bash scripts/check_lm_alignment_fixture.sh \
  runs/model_alignment/model_alignment_fixture.json
```

Without `MFT_LM_ALIGNMENT_FIXTURE`, the C++ test reports a skip. This keeps CI
independent from large local weights while making real-asset alignment one
command once a snapshot is available.

### Android Native MF Helpers

The maintained Android helpers build and run the native MF Qwen/QNLI path:

```bash
bash scripts/android/build_qwen_android.sh
bash scripts/android/stage_qwen_qnli_phone_assets.sh
bash scripts/android/run_qwen_qnli_native_phone.sh
```

Environment overrides:
- `ANDROID_NDK_ROOT`, `ANDROID_NDK_HOME`, `ANDROID_HOME`, or `ANDROID_SDK_ROOT`
  to locate Android tooling
- `ADB` to override the adb binary
- `DEVICE_ROOT` and `DEVICE_OUT_DIR` to choose phone-side asset/output paths

### `pretokenize_wikitext2_gemma.py`

Offline-tokenizes WikiText-2 for Gemma-family experiments.

```bash
python3 scripts/pretokenize_wikitext2_gemma.py
```

Outputs:
- `data/wikitext2/pretokenized_gemma/wt2_gemma_tokens.bin`
- `data/wikitext2/pretokenized_gemma/meta.json`

### Dataset Preparation Scripts

The dataset converters produce local JSONL artifacts under ignored output
directories. They do not download or bundle benchmark data into the source tree.

```bash
python3 scripts/prepare_qnli_jsonl.py --help
python3 scripts/prepare_mcq_jsonl.py --help
python3 scripts/prepare_arcc_jsonl.py --help
```

### `plot_loss_curve.py`

Plots a loss curve from a training log.

```bash
python3 scripts/plot_loss_curve.py path/to/train.log path/to/loss.png
```
